## The Micro-Tool Template (6 Sections)

Every psychological micro-tool follows this **exact** structure. The shape never changes — only the voice, metaphor, and system lens vary.

### Section 1: Immediate Insight (Hook)
**Purpose:** Instant value. No fluff.
- 1-2 short paragraphs
- Present tense
- Declarative, confident
- Feels "seen"

> *There's a part of you that operates quietly but decisively. It doesn't seek attention — it seeks impact.*

### Section 2: Naming the Pattern
**Purpose:** Cognitive clarity + emotional resonance.
- Give the insight a label
- Make it feel like something they've always known but never named
- This naming is sticky and highly shareable

> *This is a latent stabiliser pattern — the ability to hold things together when others fragment.*

### Section 3: How It Shows Up (Real Life Translation)
**Purpose:** Validation through recognition.
- 3-5 bullet points
- Concrete, everyday behaviours
- Neutral-to-positive framing
- This is where users think: "Oh… that's accurate."

### Section 4: The Cost of Not Seeing It
**Purpose:** Gentle tension without fear-mongering.
- Short
- Framed as unconscious underuse, not failure
- Creates motivation to go deeper

### Section 5: How to Work With It (Micro-Action)
**Purpose:** Empowerment, not instruction overload.
- 1-2 simple reframes or actions
- Internal, not behavioural commands

### Section 6: Expansion Bridge (Standardised CTA)
**Purpose:** Turn insight into exploration. CONSISTENT across all micro-tools.

Template:
> *This [pattern/strength/insight] doesn't exist in isolation. It expresses differently across astrology, numerology, lunar cycles, and reality codes.*

Buttons:
- 🌟 Apply to All Systems (primary CTA)
- 📚 Save to My Library
- 🧭 Explore Related Tools

## Language Rules

### Positioning
Quantum Merlin is a **personal navigation system for consciousness** — NOT a fortune-telling site.

### Shift FROM → TO
| Avoid | Use Instead |
|---|---|
| "Reading" | **Insight**, **Discovery** |
| "Prediction" | **Pattern**, **Navigation** |
| "Analysis" | **Self-mapping**, **Blueprint** |
| "Fortune" | **Alignment**, **Configuration** |
| "Tells you" | **Reveals**, **Illuminates** |
| "You should" | **You might discover**, **Consider** |

### Voice Constants (All Content)
- **Present tense** — "You ARE manifesting" not "You will manifest"
- **Declarative** — speak with certainty, not hope
- **Second person** — always "you", never "one" or "the user"
- **No disclaimers** — don't say "this is for entertainment only" inside readings
- **Inclusive** — works for any gender, age, background
- **No jargon gatekeeping** — explain mystical terms when first used

## Reading Architecture

### Content Modules

The reading system is layered:

1. **`calculations.js`** — Pure math. Returns numbers and sign names.
2. **`readings.js`** — Base interpretive text for each number/sign.
3. **`readings-extended.js`** — Extended interpretations, additional paragraphs.
4. **`deep-*.js`** (12+ modules) — Deep reading content for specialised systems:
   - `deep-astrology.js` — Sun, Moon, Rising, planetary interpretations
   - `deep-biorhythm.js` — Cycle interpretations and guidance
   - `deep-challenges.js` — Challenge number deep interpretations
   - `deep-chinese-zodiac.js` — Animal + element + compatibility
   - `deep-destiny.js` — Destiny number extended readings
   - `deep-expression.js` — Expression number meanings
   - `deep-gematria.js` — Name gematria interpretations
   - `deep-moon.js` — Moon phase and moon sign readings
   - `deep-numerology.js` — Life Path extended readings
   - `deep-pinnacles.js` — Pinnacle period interpretations
   - `deep-soul-urge.js` — Soul Urge extended meanings
   - `deep-tarot.js` — Tarot card interpretations

### Reading Generation Flow
```
User inputs birth data
  → calculations.js produces raw numbers/signs
  → readings.js provides base text for each result
  → deep-*.js adds rich paragraphs for the "full reading" version
  → Page assembles sections into formatted HTML
  → User sees personalised reading
```

### Canonical Sources
- **`ultimate/`** — Source of truth for all reading content
- **`soulblueprint/`** — Source of truth for calculations
- Other brands copy from these

## Introduction Requirements

1. Intro MUST be optional (max 1-2 sentences)
2. Skip button MUST always advance immediately
3. No blocking loaders without timeout (15s max failsafe)
4. Fail gracefully (fallback text if something breaks)

```html
<div class="intro-prompt">
    <p>This tool reveals a hidden pattern shaping your decisions. Under 60 seconds.</p>
    <div class="intro-actions">
        <button onclick="beginTool()">Begin</button>
        <button onclick="skipIntro()">Skip intro</button>
    </div>
</div>
```

## Writing New Reading Content

When creating new content for a tool or extending an existing reading:

1. **Check the existing `deep-*.js` module** — does it already cover what you need?
2. **Match the tone** to the brand (see voice variations above)
3. **Follow the 6-section template** for micro-tools
4. **Connect every insight** to at least one of the Nine Systems
5. **Test with edge cases**: Master Numbers, zero challenges, missing birth time
6. **Never leave a section empty** — always have fallback text
